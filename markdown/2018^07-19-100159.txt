# Session1. 国際的な視点から Level3/Level4/Level5自動運転を総括し、今後の動向を予測する(名古屋大学/Intel 野辺継男)
* 自動運転の定義
  * Level2: ブレーキ、ハンドリングが連携し、自動運転可能
  * Level3: オートクルーズ
  * Level4: 地図から情報を取得し、自動運転する
  * Level5: 人間が運転できる場所であればどこでも自動運転できる

* 車とICTの歴史
  * ナビゲーション搭載(2010)
    * 国外: 20%
    * 国内: 50%
      -> 日本はガラパゴス化が進んでいた
  * スマホとの連携(2010~2014)
  * ECUの搭載(2015~)
    * 運転の補助により,運転事故による割合が世界的に減少
  * ECUの発展
    * 自動ブレーキ, 位置補正(2015)
  * 自動運転の登場(2017)
    * Level3が1車種登場
  * 完全自動運転(2020)
    * Level4?
    * 人工知能時代には自動運転技術は指数関数的に進歩する

* ITと自動運転
  * 今までのようにガラパゴスな技術にしてはいけない  
    * 海外展開も可能にするべき、技術があっても海外スタンダードにならなければ  
      これからの時代はすたれていく(cloudの登場により海外展開の敷居が下がりつつある)
    * 競争と協調が必要  
      技術の中で特異な地位を確立し、深堀から他社に使用してもらえるようにする  
      (デファクトスタンダードを目指す)

* 車載化の課題
  * 車の使用年数は平均13年  
    車体差で計算能力に差がでてくる。よって、SWは考慮する必要がある
    -> これからはHW能力はどんぶり勘定で載せておく必要がある
  * 知能化における弊害
    * 必要にない情報は人間にとって不快であり、機能をoffにされる  
      (車体がラインによっているなどのナビゲーション)
      -> 必要な時に必要な情報を提示する(状況判断から警告を促す)  
         一方通行の看板とアクセルの状態を認識し、ドライバーに警告を促すなど

  * 信号などの認識
    信号などが多数ある場合どの信号が直近の信号かわからない  
    -> ADASの使用(地図情報と位置情報を連携し、直近の信号を割り出す)

* ADAS(Advanced Driver Assistance System)  
  個々の車体情報、cloud上の情報などを知見として利用可能。また随時更新されている  
  * 個々の車体から状況をcloudにあげ、それらの情報を分析し、個々の車体に情報を反映する
  * 状況認識を可能にする

* 自動運転技術
  基本はルールベース、ルールをDNNに提要する
  * アノテーション  
    センサー情報から物体を意味づけする
  * 1/10秒で計算をし直す

* ODD(Operational Design Domain)
  * 2016 自動駐車
  * 2015 交通渋滞自動運転
  * 2020
    * 高速道路: 車間距離追従, 車線キーピング, 追い越し, 入線, コンボイ走行
    * 一般道路: 障害物の回避...etc

* Teslaのインパクト(Level3は難しい)
  * カメラがトラックを認識できず死亡事故を起こした
  * Level3が仮に実現できた場合でも人間が運転意識をなくすことは非常に危険なため,  
    Level2で販売することが続く

* Level4を利用したモビリティ事業
  * Hub and Spoke(last one mile)  
    最寄駅から家までを走行(シェアリングカー)

* 質疑応答
  * 自動運転はどこが先導する?  
    -> 基本的にはプラットフォーム(ICT, 自動運転連携)をおさえた企業が先導する
  * 三次元地図をどのように収集しているのか?  
    -> 地図会社からの取得  
    -> エンドユーザーのスマホから(将来)  
    -> GoogleのGmap作成用車体から(二次元 -> 三次元地図の生成)  
    -> Teslaのように自車(販売した車体)から地図を生成していく
  * EV

# Session2. 完全自動運転ソフトウエア Autowareの概要  
  二宮 芳樹（名古屋大学 未来社会創造機構 モビリティ領域 特任教授／株式会社ティアフォー 取締役）
* モビリティサービス  
  トヨタなどはモビリティサービスで物流などにも手を伸ばす方針(amazon, pizza_hatの配送など)

* 最先端自動運転車
  Level4 Waymo(Google)

* 事故による損失
  日本: 6兆3千億

* 課題
  * 天候, 道路の条件, 交通習慣・ルールへの対応
  * 幅広い技術領域への対応(AI&データサイエンス, ロボティクス, センサ, 通信)

* 解決  
  -> オープンソース化しては?

* 一般道完全自動化運転に向けて
  * フィジカルインフラ
  * ソフトウェア解決  
  上記を統合(いいとこどりでLevel4を実現)
  * Autowareを作成
    * ディジタルインフラ(高精度地図)
      * 論理構造(ディジタル地図)
      * 物理構造(カメラからのリアル情報)
      物理構造と論理構造を照合する
    * LiDAR(3D-SCANNING LASER)  
      位置推定
    * AI

* Autowareについて
  * 3D-LiDARを含むSensor Fusion
    * 3D-LiDAR
      直接計測するためロバスト, ステレオカメラは推定問題と帰着するためロバスト性は低い
    * カラーカメラ

  * 認識手法
    * 画像認識(NDT Scan Matching)
    　ボクセル情報を関数に写像し、その関数値と現在のリアル情報(LiDARでとれた点情報)
      との距離を計測し、マッチングをしている
    * 状況認識・判断
      DNNベース(YOLO2, SSD, セマンティックセグメンテーション, セマンティックセグメンテーション)  

  * 提供リソース
    * データ
    * アノテーション

  * 対抗OSS
    ApolloAuto/apollo

* 質疑応答
  * LiDARのコストはどの程度?  
    -> 答えられない、ただし量産化すると安価になる予定(基本的には人件費)
  * 日本はどの程度自動運転で遅れているのか?  
    自動運転の状況によるため、定量的に遅延を表せないが、自動運転に使用している
    技術(DNN, LiDAR)はほぼすべて外国産  
    また、自動運転の実地試行回数の桁が違う
  * 事故の際の被害者に対する優先度づけはどうしている?
    -> 結論はでていない、調査ではマジョリティーを優先度高にすべきとでているが  
       助手席が被害者になる場合は優先度がかわるなど、まだまだ課題はある

# Session3. 完全自動運転が生み出す社会変化とSynergic Mobilityの創出
  河口 信夫（名古屋大学 未来社会創造機構 教授）

* シェアからシナジーの世界へ
  * パラダイムシフトによる社会の超高効率化  
    -> 多種のサービスを同時に実現する技術  
       (車: 人の移動, 遠隔検診, 宅配..etc)

  * Owner Cars(個人所有車)とMaas(Mobility as a Service)

* サブシステムと外部環境  
  多種のサービスカーとそれらから得られる外部環境情報を連携する必要がある  
  -> 動くスマートフォンを目指す

* マーケット
  自動運転をとりまくマーケットは情報を統合したマーケットサービスが必要  
  ホテル予約のように情報源が乱立し、まとめサイトなどがでることをふせぐ
  * 適切なルーティングを提供する
  * 過去の需要データを活用新規参画者が活用可能

* 自動運転と自動運転技術の応用
  * LiDARの使用して路面調査を行う(路面調査は車両と調査費のみで数億かかる)  
    -> LiDARを使用すれば数百分の一で可能
